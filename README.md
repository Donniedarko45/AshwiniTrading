# Ashwini Trading Academy - Secure Payment System

This repository contains the complete, production-ready payment module for the Ashwini Trading Academy website. The module is built using **React (Vite, TypeScript, Tailwind CSS v4, shadcn/ui)** on the frontend, **Node.js (Express, TypeScript, Prisma ORM)** on the backend, and is integrated with the **Cashfree Payment Gateway** and **PostgreSQL**.

---

## Features

- **End-to-End Payment Lifecycle**: From course selection, checkout, and gateway redirection, to status polling and redirection.
- **Robust Security**: 
  - Cryptographic HMAC-SHA256 signature verification for Cashfree webhooks.
  - Server-side double-verification of all transaction states (never trust the client).
  - Express Rate Limiting on checkout, coupon, and payment retry endpoints.
  - Protection against duplicate purchases (blocks buying a course a user already owns).
- **Premium User Experience**:
  - Itemized pricing breakdowns extracting 18% inclusive GST (Taxable Base + CGST + SGST).
  - Real-time API-driven coupon code validation (`WELCOME10` for 10% off, `EARLYBIRD` for flat ₹1500 off).
  - Self-healing retry payments (generates a fresh gateway session for failed attempts).
  - Auto-polling loading screen during transaction verification.
  - Custom client-side GST Invoice Generator (printable/downloadable receipt).

---

## Tech Stack

- **Frontend**: React 19, Vite, TypeScript, Tailwind CSS v4, shadcn/ui, Lucide Icons, Motion.
- **Backend**: Node.js, Express.js, TypeScript (`tsx` runner), CORS, Express Rate Limit.
- **Database**: PostgreSQL database instance, Prisma ORM 7.x (with modern `prisma.config.ts` configuration).
- **Payment Gateway**: Cashfree PG (Web Checkout SDK v3).

## Local Setup Instructions

### 1. Prerequisites
- **Node.js** (v18 or higher recommended)
- **PostgreSQL** running locally or hosted (e.g. Supabase, Neon)

### 2. Install Dependencies
In the root directory of the project, run:
```bash
npm install
```

### 3. Environment Configuration
Copy the environment template and fill in your credentials:
```bash
cp .env.example .env
```
Open `.env` and configure:
- `DATABASE_URL`: Your PostgreSQL connection string.
- `CF_CLIENT_ID`: Your Cashfree App/Client ID (found in Merchant Dashboard).
- `CF_CLIENT_SECRET`: Your Cashfree Client Secret.
- `CF_ENVIRONMENT`: Set to `sandbox` for local testing.

### 4. Initialize the Database
Generate the Prisma 7 client and push the schema to your PostgreSQL database:
```bash
npx prisma generate
npx prisma db push
```

### 5. Seed Database Records
Seed the database with all 9 academy programs, 4 specialized workshop courses, and active discount coupons:
```bash
npx tsx prisma/seed.ts
```

### 6. Run the Application Locally

#### Start the Backend Server (Port 5000)
Run the Express API server in a separate terminal:
```bash
npx tsx server.ts
```

#### Start the Frontend Dev Server (Port 3000)
Run the Vite frontend in another terminal:
```bash
npm run dev
```
Open your browser and navigate to `http://localhost:3000`. You can click **Enroll Now** on any course card to test the payment journey!

## Testing the Payment Flow

### 1. Run Automated Test Suite
We have provided an isolated, cryptographic business-logic test suite in the `scratch/` folder. It tests coupon calculations, GST extraction, duplicate checks, and webhook signature validation:
```bash
npx tsx scratch/verify_payment_system.ts
```
*(All 17 core assertions should output `[PASS]`)*

### 2. Testing Webhooks Locally
Since Cashfree sends webhook events asynchronously to your backend, your server must be publicly accessible.
1. Download and run a tunneling service like **ngrok**:
   ```bash
   ngrok http 5000
   ```
2. Copy the forwarding HTTPS URL generated by ngrok (e.g. `https://a1b2-c3d4.ngrok.app`).
3. Set this URL as the `APP_URL` in your `.env` file, and restart the backend server:
   ```env
   APP_URL="https://a1b2-c3d4.ngrok.app"
   ```
4. Perform a transaction in Sandbox checkout, select "Simulate Success", and watch your backend terminal logs. You will see:
   - `Webhook Received: Event=PAYMENT_SUCCESS...`
   - `Order ORD_XXX marked as PAID. Transaction: CF_...`

## Production Deployment Instructions

Follow these instructions to deploy this module to a production cloud environment (e.g. Render, Railway, AWS, or DigitalOcean).

### 1. Database Setup
- Provision a production-ready PostgreSQL instance (e.g. AWS RDS, Supabase, Neon, or GCP Cloud SQL).
- Obtain the connection string and configure it as `DATABASE_URL` in your production environment variables.

### 2. Gateway Production Credentials
- Go to the [Cashfree Merchant Dashboard](https://merchant.cashfree.com/merchant) and activate your account.
- Generate your **Production API Keys** (Client ID and Client Secret).
- Ensure your production domain is whitelisted in the Cashfree dashboard under **Whitelists**.

### 3. Deploying the Backend (Express)
- **Environment Variables**: Define all variables from `.env.example` in your hosting dashboard:
  - `PORT=8080` (or leave blank for platform default)
  - `APP_URL=https://yourdomain.com` (Your production frontend URL)
  - `DATABASE_URL=postgresql://...` (Production database connection string)
  - `CF_CLIENT_ID=...` (Production App ID)
  - `CF_CLIENT_SECRET=...` (Production Secret Key)
  - `CF_ENVIRONMENT=production`
- **Build & Start Commands**:
  - Build Command: `npm run build && npx prisma generate`
  - Start Command: `npx prisma db push && npx tsx server.ts` (Runs migrations/syncs and starts the server)

### 4. Deploying the Frontend (Vite Static Build)
Vite builds your frontend into static HTML, CSS, and JS.
- Ensure that the frontend is configured to call your production backend API. The development proxy configuration in `vite.config.ts` is only active during local development.
- For production, ensure the frontend calls the deployed backend API (e.g., set an environment variable `VITE_API_URL` to your backend domain, and fetch from `${VITE_API_URL}/api/...` instead of `/api/...`, or configure your web server/reverse proxy like Nginx to route `/api/*` to the Express backend port).
- Build the production bundle:
  ```bash
  npm run build
  ```
- Upload the `dist/` directory to a static site host (e.g. Vercel, Netlify, Cloudflare Pages, or AWS S3).
